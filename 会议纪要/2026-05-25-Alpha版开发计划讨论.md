## 最终确认决策（许先生拍板）

1. **LLM：** DeepSeek（API Key 已提供）
2. **角色选取：** 方案A — LLM根据议题复杂度自动选角，支持3-10个角色与会
3. **会议总结：** DAG末尾加总结step，由LLM综合所有观点输出结论
4. **追问机制：** 会议结束后追问（非实时打断），保持POC设计的最多2轮

## Alpha 完整用户流程

```
用户输入主题
 ↓
LLM 自动选角（3-10个，根据复杂度决定）
 ↓
第一轮讨论（角色逐个发言，WebSocket 逐条推送）
 ↓
第二轮交叉讨论（角色看到其他人观点后回应）
 ↓
总结 step（综合结论 + 分歧 + 行动建议）
 ↓
用户可追问（会议结束后，最多2轮）
```

## 技术可行性确认

- agency-orchestrator v0.6.17 已安装，角色库完整（200+角色，15类别）
- AO 原生支持 DeepSeek：`createConnector({ provider: 'deepseek' })`
- POC 已用 `buildDAG` + `executeDAG`，只需替换 echo connector 为真实 LLM connector
- 角色选取：AO 不自带选角，需一次额外 DeepSeek 调用（成本约0.01元/次）
- 会议总结：DAG 末尾加一个 depends_on 所有角色的 step 即可
- `normalizeRoles` 从强制3个改为支持2-10个
