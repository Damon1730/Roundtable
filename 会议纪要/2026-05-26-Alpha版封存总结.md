## 关键技术方案

### 架构

```
前端 (React 19 + Vite 7)
  ↕ WebSocket + HTTP
后端 (Node.js + ws + better-sqlite3)
  ↕ agency-orchestrator DAG
DeepSeek API (选角 + 角色发言 + 总结)
```

### 核心流程

1. 用户输入主题 → WS `run_roundtable`
2. DeepSeek 自动选角（3-10个，从 211 个角色中选）
3. DAG 并行执行（concurrency: 3），角色独立发言
4. 总结 step 综合所有观点 → 结论/分歧/行动建议
5. 支持追问（最多 2 轮）

### Prompt 工程（T18 修复后的最终方案）

- **system prompt** = 角色 .md 全文（由 ensureAgentFiles 写入 .roundtable-agents/）
- **user message (task)** = `请以${role.name}的专业视角，围绕以下主题发表你的观点和建议：\n主题：${topic}\n要求：直接针对主题给出专业分析，不要复述你的身份设定。`

修复前的错误做法：task = `以${role.name}身份发言：${role.prompt}`（把 9KB 角色设定塞进 user message，LLM 只会复述人设）

### 容错机制

- 单角色超时 60s，不阻塞其他角色
- `withTimeout` 包裹每个 LLM 调用
- mock 模式用 `await new Promise(() => {})` 模拟永不返回

### 前端 404 修复（T19）

- `loadMeeting(meetingId)` 仅在历史列表中存在该会议时才发 HTTP 请求
- 新建会议（前端生成 ID，服务端尚未创建记录）不触发无效请求

## 踩坑记录

1. **Worker APITimeoutError** — 看板 worker 自身用麦冬 API 作为 LLM 大脑，与项目代码用的 DeepSeek 是两层。麦冬超时导致 worker 崩溃，需设 `dispatch_max: 2` 限制并发。
2. **Windows git-bash 后台进程** — 普通 `background: true` 报 "stdin is not a tty"，必须用 `pty: true`。
3. **agency-agents-zh 加载路径** — 本地 git clone 的目录是空的（只有 .git），实际角色从 `node_modules/agency-agents-zh/` 加载（npm 依赖）。
4. **Prompt 工程** — 角色 .md 全文绝不能放进 user message，必须作为 system prompt。否则 LLM 把人设当输入复述。
