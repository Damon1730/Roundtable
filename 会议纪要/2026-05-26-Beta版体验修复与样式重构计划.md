# Beta 版体验修复与样式重构计划

**日期：** 2026-05-26
**分支：** `Codex`
**阶段：** Beta 前置体验修复完成，下一迭代进入样式重构
**记录时间：** 2026-05-26 09:03

---

## 背景

本轮工作基于 Alpha 已验收能力继续推进：自动选角、真实 DeepSeek 发言、两轮圆桌讨论、会议总结和追问能力已经具备。Codex 分支此前的节奏优化出现了三个核心问题：

1. 法律/裁员类问题会错误选到前端、后端、测试或产品、内容、开发等无关角色。
2. 角色发言有时不围绕用户主题，而是复述角色设定或跑到通用业务话题。
3. 前端打字机与历史回看状态相互覆盖，导致内容重复播放、人员分配消息消失、总结布局跳动。

本轮目标是先恢复“圆桌会议能正确讨论用户主题”的产品底线，再为 Beta 样式重构做准备。

---

## 思考内容

### 1. 正确性优先于视觉节奏

圆桌会议的核心不是“文字慢慢出现”，而是系统能根据议题选择合适角色，并让角色围绕议题讨论。如果选角和发言内容错误，再好的打字机效果也只是掩盖问题。因此本轮先修后端选角与 prompt 约束，再修前端播放体验。

### 2. Alpha 行为作为基线

Alpha 封存时已经确认：

- system prompt 使用角色 `.md` 全文。
- user message 只放任务和主题。
- 自动选角从 `agency-agents-zh` 角色库中选择。
- 所有角色必须围绕主题讨论，不能复述人设。

Codex 分支的修复不再沿用废弃实现，而是回到 Alpha 基线后重新引入两轮讨论与播放队列。

### 3. 概念图可作为视觉方向，但不能直接照搬

`设计稿/AI圆桌会议室-深色概念图.png` 更像桌面端专业工作台方向，适合作为 Beta 样式重构参考。当前功能可以支撑三栏布局、阶段条、发言卡片和右侧总结，但完整复刻需要补充更细的数据结构，例如 `stage`、`round`、`replyTo`、`roleMeta`、`summarySections`。

## 实际开发内容

### 后端

1. 修复真实 DeepSeek 选角解析：
   - 支持按 `category/filename` 精确匹配。
   - 支持按角色名、文件名、agent id 兜底匹配。
   - 解析失败时不再直接中断会议。

2. 增加主题适配保护：
   - 对裁员、劳动、合同、合规、法律、风险类议题，若模型选角不含足够法务/HR/风险/财务角色，则强制回退到关键词匹配角色。
   - 避免法律问题回退到 POC 时代的前端、后端、测试角色。

3. 修复角色库路径：
   - 本地 `agency-agents-zh/` 为空目录时不再误用。
   - 只有存在 `AGENT-LIST.md` 时才使用本地角色库，否则使用 `node_modules/agency-agents-zh/`。

4. 强化发言任务 prompt：
   - 第一句必须直接回应用户议题。
   - 法律/裁员类问题必须围绕劳动合同解除、补偿、程序、证据留痕、社保工资结算、合规审批和争议风险。
   - 禁止编造与议题无关的业务数据、审计数据或案例。

5. 保留两轮会议流程：
   - 第一轮短发言。
   - 第二轮基于第一轮 transcript 回应其他角色。
   - 总结基于数据库中已经成功写入的发言生成。

### 前端

1. 修复人员分配消息：
   - `roles_selected` 不再直接插入页面，而是进入播放队列。
   - 人员分配消息也使用打字机效果。
   - 会议结束后不再因为历史回看刷新而消失。

2. 调整打字机速度：
   - 从 `45ms/字` 调整为 `60ms/字`。
   - 目标是略快于常人阅读速度，降低“闪太快”的感觉。

3. 修复布局跳动：
   - 系统消息固定左侧。
   - 总结消息固定左侧。
   - 错误消息固定左侧。
   - 不再因奇偶消息序号导致总结在会议结束后从右侧跳到左侧。

4. 修复实时播放与历史加载覆盖：
   - 会议结束后不再立即把当前实时会议切换为历史回看。
   - 保留实时播放中的系统选角消息和打字机状态。

### 测试

已验证：

- `node --check server/index.js`
- `npm run build`
- `test-e2e.js`：110 项断言全部通过

真实 DeepSeek 验收题：

```text
公司准备裁员，怎样降低劳动法律风险？
```

实际选角已能落到合同、制度、招聘、绩效、财务、法务、风险等相关角色，首条发言能直接讨论裁员程序、经济补偿、违法解除、证据留痕等劳动法风险。

## 即将开始的 Beta 样式重构计划

下一迭代进入 Beta 视觉与结构重构，参考深色概念稿，但不直接照搬。

### 计划目标

1. 将当前浅色单栏会议页重构为深色专业工作台。
2. 建立三栏结构：
   - 左侧：会议历史、新会议入口、用户信息。
   - 中间：会议主题、阶段条、时间线发言卡片、追问输入。
   - 右侧：会议总结、共识、分歧、行动建议、风险提示。
3. 将会议过程从“消息列表”升级为“阶段化会议流”。
4. 为后续小程序/H5 Beta 版抽象可复用的数据结构和组件边界。

### 需要补充的数据结构

- `stage`：选角中、第一轮、第二轮、总结、追问。
- `round`：第几轮发言。
- `replyToRoleId` / `replyToMessageId`：第二轮回应对象。
- `roleMeta`：角色分类、头像、专业标签、颜色。
- `summarySections`：核心结论、共识、分歧、行动建议、风险提示。

### 暂不优先实现

- 附件上传。
- `@` 提及专家的定向追问。
- PDF/Word 导出。
- 真实头像生成。
- 移动端小程序适配。

这些能力需要后端协议、存储和权限模型进一步设计，不宜在样式重构第一步强行加入。

---

## 当前结论

Beta 下一步不是继续微调现有浅色 UI，而是基于概念稿做一次结构化重构。重构前必须先固化消息协议，否则页面只能靠前端猜测阶段、轮次、回应关系和总结结构，后续会越来越难维护。
